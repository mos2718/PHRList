FHIR and DICOMWeb imaging workflow

  • 1. 開立影像檢查處方 (Ordering)
  • 2. 影像檢查排程(Scheduling)
  • 3. 儀器造影 (Imaging acquisition)
  • 4. 打報告 (Reporting)

範例資料與程式(examples)

1. 開立影像檢查處方 (Post serviceRequest)

2. 影像檢查排程(Scheduling)

  • 2.1 取得影像檢查處方 (Get serviceRequest)
  • 2.2 預約檢查及報到(Post appointment and encounter)
  • 2.3 依據處方及排程資訊，產生工作清單, 注 1 (Post or get serviceRequest and encounter. Create DICOM MWL or UPS)
注 1: 現行 PACS server 及儀器支援 FHIR 及 DICOMWeb 的程度不一， 可能轉換介面架在server 或儀器端。 介面系統因處於server 或儀器端，可能扮演 HTTPServer 或 client 及 DICOM MWL server。 RIS order filler 系統可能修改檢查項目

3. 儀器造影 (Imaging acquisition)

  • 3.1 取得工作清單, 注 2 (Retrieve MWL or Get UPS)
  • 3.2 上傳影像(STOW image store)
  • 3.3 上傳影像清單, 注 3 (Post imagingStudy)
  • 3.4 調閱影像清單，確認儲存完成 (Get imagingStudy)
注 2: MWL 及 Get UPS 內容可與 serviceRequest and encounter 對應， 儀器或介面程式可從 MWL, UPS,serviceRequest 取得檢查項目。
注 3: 可從 DICOMWeb server 或 FHIR server 確認影像儲存完成。 imagingStudy 也可用於 QA QC。

4. 打報告 (Reporting)

  • 4.1 查詢要打報告的病人 (Get encountered patient)
  • 4.2 調閱病人歷次檢查影像影像清單(Get imagingStudy)
  • 4.3 調閱影像(WADO-RS retrieve)
  • 4.4 產生影像標記、發現、及報告(Post annotation and finding observation and diagnosticReport)
